Three Things Leadership Needs to Know This Month

🟢 German Books – Primary Path to 13M

German genealogical books have emerged as the primary source for achieving the 13M Automated Tree Building goal. Content Archaeology has already loaded 47M names into user trees, and sampling shows deep genealogies with strong connections to US diaspora members. Jon Morrey estimates books alone could yield the entire 13M+ target.

Status: OGC issued "proceed" directive June 23, with remaining restrictions expected removed by end of August. Technology work continues on ATB quality improvements and Reference Tree buildout, with November ATB activation targeted for delivery.

🟡 Critical Timeline Pressure on Reference Tree Delivery

ATC (Reference Tree ordinance consumption) investigation in June revealed significant complexity in ordinance system integration. Team discovered new technical requirements: ordinance registry/snapshot database for ephemeral PIDs, ordinance approval required (OAR) flag logic, long-lived ID requirements for temple recording.

Decision Point: Mid-September 2026 established as go/no-go decision point for executing Plan B – bulk copying 13M high-confidence names into Family Tree if end-to-end ATB→ATC system isn't ready by year-end. One engineer has been allocated to develop emergency plan in parallel. (See Big Rocks → Rock #4 for technical details)

🟢 Measurement and Attribution Systems Advancing

Manual ORN inventory reporting in Databricks (Lyle Toronto/John Sumsion) targeting July delivery. PoC demonstrated capability: 1.7B person table with baptism qualification status, multi-value disqualification reasons, country/state granularity.

Timeline: Automated incremental reporting targeted for September. Rock #6 (New ORN Reporting Data) remains on track to provide accurate, attributed reporting by end of Q3, including Lever 3 (connect to members) attribution. Critical dependency: Platform SOC Developer Platform team priority on P20 Databricks production infrastructure.

Key Update

Execution Team 1: User Activity

User Activity — 6.32M YTD of 23M goal (27%)
Coordinates: 1 supply source (Members & genealogists building trees)
REVISED TO 14M
▼

YTD / Goal: 6.32M of 23M (27%)

Anticipated Results: 14M (revised from 23M)

User Activity is at 6.32M YTD — 27% of the 23M annual goal. Goal revised to 14M anticipated results based on actual performance data showing 0.85M/month average vs. 2M/month originally assumed.

Two structural factors explain the gap:

  • Record visibility constraints: Visible records have remained flat at 4.1B for the past 10+ months, largely due to the Themis rights changeover restricting more records than anticipated. This is believed to be an important limiter of user activity, as hint generation is automatic for newly published records.
  • Prior-year distortion: Non-replicable 2025 measurement spikes inflated year-over-year comparisons and the 2026 goal, creating an artificial performance gap.

🔴 Rock #4: ORN Qualification & Release from Reference Tree
Status: RISKS EXIST
▼

Purpose: Deliver infrastructure and interfaces that enable patrons to receive ORNs from Reference Tree, integrating with existing ordinance systems.

Note: Reference Tree and ORN qualification work have been combined into this single rock.

Major Complexity Uncovered:

ATC (Reference Tree Consumption) investigation revealed significant ordinance system integration complexity:

  • Ordinance approval required (OAR) flag logic not previously scoped
  • Long-lived ID requirements for temple recording
  • PID-history tracking requirement for ordinance system integration
  • Grove graph database (300K queries/sec) selected to replace Tiger Graph PoC failure (7 queries/sec insufficient)

Go/No-Go Decision: Mid-September 2026 established as decision point for executing Plan B — bulk copying 13M high-confidence names into Family Tree if end-to-end ATB→ATC system isn't ready by year-end.

Active Work:

  • Determining ORN qualification status from Reference Tree, including related-to-member
  • Implementing solutions to support graph queries for the Reference Tree (Grove database)
  • Exploring integration points & architecture changes with the MTE SOC
  • One engineer allocated to develop Plan B emergency path in parallel

Resource Reallocation: ATBX team (5 devs, 2 QA) charter complete; engineers temporarily redistributed through Dec 2026: 3 to Databricks/data platform work, 1-2 to ATC or Tree Foundation.
